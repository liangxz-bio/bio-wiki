# CroCoDeEL：无需对照准确检测宏基因组数据中的交叉样本污染

> **来源：** Nature Communications | 原创：小包子学演化

---

## 摘要

宏基因组测序为微生物群落提供了深入见解，但可能受到技术偏倚的影响，包括**交叉样本污染**。现有检测方法依赖阴性对照，但存在以下局限：

- 对照使用不足
- 无法检测非对照样本之间的交叉污染
- 基于菌株水平的方法无法区分污染与自然菌株共享
- 灵敏度不足

为此，研究团队推出了 **CroCoDeEL**（CROss-sample COntamination DEtection and Estimation of its Level），一种用于检测和定量交叉样本污染的决策支持工具。

**关键词：** 宏基因组学、质量控制、交叉样本污染、监督分类

## 引言

### 背景

鸟枪法宏基因组测序彻底改变了微生物学研究，但在不同步骤中会受到技术和实验偏倚的影响。污染分为两类：

1. **外源污染**：来自采样环境、实验室台面、DNA提取试剂盒等
2. **交叉样本污染**（孔间渗漏）：在同时处理的样本之间意外转移物质（"splashome"）

### 交叉样本污染的成因

- 人为错误或机器人操作失误
- 重复使用移液器吸头
- 移液渗漏、封板时形成的微液滴
- 振荡过程中的飞溅
- 测序过程中的**标签跳跃**

### 现有方法的局限性

| 方法 | 局限性 |
|------|--------|
| 阴性对照 | 无法揭示仅发生在真实样本之间的污染事件 |
| SCRuB | 非设计用于检测对照之外的交叉样本污染 |
| Recentrifuge | 要求污染源也影响了阴性对照（实践中常不成立） |
| Lou等人菌株共享法 | 难以区分污染与自然菌株共享；低水平污染检测灵敏度不足；计算量大 |

## 结果

### 1. 物种丰度谱中的特定模式与交叉样本污染相关

**核心原理：** 交叉样本污染可被视为样本稀释，污染特异性物种的相对丰度在受污染样本和污染源之间呈**比例关系**，比例系数等于污染率。

在对数尺度散点图中，这一现象表现为一条"**污染线**"：
- 污染前：两样本物种丰度谱无明显线性模式
- 污染后：污染特异性物种（橙色点）丰度呈明显线性模式，方程为 `污染率 × 污染源丰度 = 受污染样本丰度`

**实验验证：** 通过将两份粪便样本 MQB 068 和 MQB 095 按约 90:10 比例混合创建人为污染样本 MQB 095l2，证实了污染线在真实宏基因组数据中可被观察到。

**半模拟数据集：** 包含 13,330 个样本对，其中 44% 为污染事件，验证集精确率和召回率均超过 **0.99**。

---

### 2. CroCoDeEL 算法流程（图2）

```
输入：两个样本的对数转换物种丰度谱

步骤1：选择污染线候选物种
  └─ 筛选在污染源中更丰富、左上象限中少有其他物种的物种

步骤2：使用 RANSAC 回归器检测潜在污染线
  └─ 估计线偏移参数，将候选物种分为内点/外点
  └─ 若内点物种 < 5 个，判定为未受污染

步骤3：提取10个数值特征（描述污染模式）

步骤4：输入预训练随机森林模型（1000棵树）进行分类
  └─ 置信度评分 > 0.5 → 标记为受污染，估计污染率
  └─ 置信度评分 ≤ 0.5 → 判定为未受污染

输出：污染判定 + 污染率估计 + 图形报告
```

**10个特征（按SHAP值重要性排序）：**

| 特征 | 含义 |
|------|------|
| 内点数量 | 构成污染线的物种数量 |
| 线紧致性 | 内点到5个最近邻的平均欧氏距离 |
| 高丰度物种缺失度 | 污染源中高丰度物种是否意外缺失于受污染样本 |
| 正交残差 | 内点到污染线的平均正交残差（线的离散程度） |
| 内点比例 | 内点数与共检测物种数之比 |
| 低丰度内点均值 | 最不丰富10%内点在污染源中的平均丰度 |
| 线上物种比例 | 位于污染线之上的物种比例 |
| 线上物种数量 | 位于污染线之上的物种数量 |
| 线延展性 | 内点到5个最远邻的平均欧氏距离 |
| Spearman相关系数 | 两样本间的秩相关系数 |

### 3. 在真实宏基因组数据中的性能评估

**基准测试结果（表1）：**

| 指标 | 平均值 |
|------|--------|
| Matthews相关系数 | **0.69** |
| 召回率 | **95%** |
| 特异性 | **50%** |

**假阴性分析：**
- 第一类：污染率极低，人类专家置信度有限
- 第二类：高水平污染但模式模糊（如"级联污染"场景）

**假阳性分析：**
- 大多数假阳性估计污染水平极低（平均 0.26%）
- 置信度评分显著低于真阳性（平均概率：0.73 vs 0.92）
- 跨数据集验证：140,972个样本对中仅检测到 **5个假阳性**

**计算资源：**
- ~100个样本：几分钟内完成
- 运行时间随样本数量呈**二次方增长**
- CPU可扩展性效率约 **0.85**
- 内存使用量低，可在标准台式计算机上运行

---

### 4. 测序深度、污染率和分类工具的影响

**关键发现：**

| 条件 | CroCoDeEL检测率 |
|------|----------------|
| 污染率20%，1M读段，Meteor2 | **100%** |
| 污染率2%，10M读段，Meteor2 | **92%** |
| 污染率2%，1M读段，Meteor2 | **40%** |
| 污染率0.5%，10M读段，Meteor2 | **76%** |
| 污染率0.5%，10M读段，Sylph | **4%** |
| 污染率0.5%，10M读段，MetaPhlAn4 | **0%** |

> ⚠️ **重要结论：** CroCoDeEL的灵敏度强烈依赖于所使用的分类分析工具。**Meteor2** 在低测序深度和低污染率下表现最优，MetaPhlAn4 系统性低估次优势物种丰度，显著影响检测能力。

---

### 5. 与菌株共享方法的比较（板P3案例）

| 方法 | 识别受污染样本数 |
|------|----------------|
| Lou等人菌株共享法 | **2个** |
| CroCoDeEL（人工验证后） | **12个**（16个污染事件） |

**CroCoDeEL的额外发现：**
- 9个污染案例发生在**相邻孔**样本之间
- 识别了纵向研究中同一婴儿不同时间点之间的污染
- 识别了母婴之间的污染（如58D7被58M污染）
- 识别了双胞胎之间的污染（60D382污染58D382）

---

### 6. 未被检测到的污染对研究结果的影响

#### 案例一：Lou等人婴儿微生物定植研究（板P2）

- CroCoDeEL识别了8个此前未被检测到的污染事件
- 婴儿粪便样本57D8被母亲样本57M以 **23%** 的比率污染
- **结论：** 原研究报告的"母体菌株垂直传播"很可能是交叉样本污染的假象

#### 案例二：Ferretti等人研究

- 182份样本中识别出 **48个**交叉污染样本
- **80%**（45/56）被报告为短暂存在的微生物可能是污染结果
- 移除受污染样本后，t1时间点的高alpha多样性不再显著（p值从<0.005变为>0.12）

#### 案例三：TwinsUK队列（1004份粪便样本）

- 识别 **202个**受污染样本，其中176个与相同的8个污染源相关
- 受污染样本 vs 未受污染样本：
  - Bray-Curtis相异性：**0.40 vs 0.53**（p<0.0001）
  - 物种丰富度：**425 vs 303**（p<0.0001）
  - **32%**（440/1382）的物种在受污染样本中流行率显著更高

## 讨论

### 主要优势

1. **无需阴性对照**，仅依赖物种丰度谱
2. **无需板布局信息**，适用范围广
3. 能检测**非相邻样本**之间的交叉污染
4. 可检测低至 **0.1%** 的细微污染
5. 提供**污染率估计**和直观图形报告
6. 计算效率高，适合大规模数据集

### 局限性

- 可能错过污染线不明显的复杂污染场景
- 样本固有相似性高时可能产生假阳性（如同笼动物、短时间间隔纵向样本）
- 设计为**决策支持工具**，非确定性分类系统，需结合人工审核

### 应用建议

> CroCoDeEL 并非取代传统对照，而是**补充**现有方法，填补检测非对照样本间交叉污染的方法学空白。

**污染严重程度的影响因素：**
- Alpha多样性分析：1%污染率可显著夸大微生物丰富度（引入数十个虚假物种）
- Shannon多样性指数：受影响较小（污染物通常以低丰度存在）
- 临床环境：风险最为严重（每个样本单独分析，虚假病原体发现可损害诊断）

## 方法

### 伦理

研究方案经法国人员保护委员会（CPP Île-de-France 10）批准，编号 **2021-A02873-38**，遵循《赫尔辛基宣言》原则，所有参与者提供书面知情同意。

### 数据预处理

```
Illumina数据：fastp (v0.23.4)
  ├─ 去除测序接头
  ├─ 修剪/丢弃低质量读段
  └─ 丢弃 < 60bp 的读段

Ion Torrent数据：AlienTrimmer (v2.0)

人类基因组去除：Bowtie2 (v2.5.4) → T2T CHM13v2.0
```

### 分类分析工具

| 工具 | 版本 | 数据库 |
|------|------|--------|
| **Meteor2**（主要） | v2.0.14 | IGC2（1040万基因，1990个MGS） |
| Sylph | v0.6.0 | 基因组分类数据库 r214 |
| MetaPhlAn4 | v4.1.0 | vJun23数据库 |

### 半模拟数据集生成

**两种污染模拟策略：**

1. **基于读段的子采样**（seqtk v1.4r122）
   - 通用性强，支持多种分类工具
   - 生成中间文件较大

2. **基于基因计数的稀疏化**
   - 计算速度更快
   - 仅限于Meteor工具

**训练数据集规模：**
- 总计：13,330个样本对（7,480个未受污染 + 5,850个受污染）
- 来源：11个独立队列，15,203个样本
- 污染类型分布：肠道→肠道 72.3%，肠道→口腔 14.5%，口腔→肠道 13.2%
- 训练/测试集划分：**70% / 30%**

### 实现环境

```python
# 主要依赖库
- Python 3
- Pandas      # 数据框管理
- scikit-learn # 随机森林 & RANSAC算法
- Matplotlib  # 散点图生成
- multiprocessing # 并行处理
```

**基准测试环境：** 两个 Intel Xeon CPU E5-2680（2.70 GHz，每个8个物理核心），256 GB RAM
