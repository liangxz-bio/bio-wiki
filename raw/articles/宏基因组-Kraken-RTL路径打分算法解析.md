# 宏基因组 | 通过Kraken学习RTL路径打分

> **作者：** Ethan Ethan
> **来源：** MetagenomeBioin（微信公众号）
> **原文：** https://mp.weixin.qq.com/s/JB_ygjU9IEBhu_uiOgWvlg

---

## 2.1 问题背景

一条 100 bp 的 read 有 70 个 31-mer。每个 k-mer 查数据库后得到一个分类单元（taxon）。但这 70 个 k-mer 给出的"意见"**可能不一致**——比如 60 个指向物种 A，5 个指向物种 B，3 个指向属 X，2 个查不到。

我们需要一种方法来**综合这些不一致的投票**，给出最终分类。

---

## 2.2 形式化定义

**输入：**

- 一条序列 $S$
- $K(S)$：$S$ 中所有 k-mer 的集合
- 每个 k-mer 通过数据库查询得到一个 LCA 分类单元
- 全局分类树（taxonomy tree）$\mathcal{T}$：从根节点（"生命"）到各物种的完整树形结构

**处理步骤：**

1. **第一步：构建分类树（classification tree）**
2. **第二步：给节点赋权重**
3. **第三步：RTL 路径打分**
4. **第四步：选出最终分类**

---

## 2.3 用一个完整例子走一遍

假设分类树长这样（数字是权重）：

```
          Root (w=0)
            |
         Domain
            |
          Phylum
            |
          Class
            |
          Order
            |
          Family
            |
          Genus X       Genus Y
            |               |
         Species A       Species B
```

每个节点的权重 = 直接映射到该节点的 k-mer 数量。

**RTL（Root To Leaf）路径打分：** 从根到每个叶节点（候选分类），将路径上所有节点的权重**累加**，得到该叶节点的得分。

## 2.4 为什么 RTL 路径打分是合理的？

这个设计体现了一个巧妙的原则：**越具体的层级获得的支持越重要，但高层级的广泛支持也会被累加进去。**

考虑两种场景：

**场景一：证据集中在特定物种**
- 大量 k-mer 直接指向物种 A → 物种 A 节点权重高
- 其路径得分 = 物种A权重 + 属权重 + 科权重 + ... → 得分远高于其他物种

**场景二：证据分散在属级**
- k-mer 只能确定到属 X，无法区分属内物种
- 属 X 节点获得高权重
- 属 X 下所有物种的路径都会累加这个属级权重
- → 具有额外物种级证据的物种得分更高，其余平分属级支持

## 2.5 和简单投票法的对比

你可能会问：为什么不直接数每个物种获得了多少票（k-mer），取最多票的物种？

简单投票法的问题在于它**不考虑分类层级**。假设有这样的情况：

- 10 个 k-mer → 物种 A
- 8 个 k-mer → 属 X（物种 A 所在的属，但没法定到种）
- 12 个 k-mer → 物种 B（另一个属 Y）

**简单投票结果：** 物种 B 胜出（12 > 10）。

**RTL 打分结果：**

- 通往物种 A 的路径得分 = 10（种）+ 8（属 X）+ 上层 = **18+**
- 通往物种 B 的路径得分 = 12 + 上层

**→ 物种 A 胜出！**

因为那 8 个只能定到属 X 的 k-mer，虽然没法确认是物种 A，但它们提供了**间接支持**——至少它们证实了序列来自属 X，而物种 A 正属于属 X。RTL 路径打分把这种"不同精度层级的证据"优雅地整合在了一起。

## 核心概念总结

| 概念 | 说明 |
|------|------|
| **k-mer 投票** | 每个 k-mer 查 LCA 数据库得到一个分类节点 |
| **节点权重** | 直接映射到该节点的 k-mer 数量 |
| **RTL 路径打分** | 从根到叶节点，累加路径上所有节点权重 |
| **最终分类** | 选取 RTL 得分最高的叶节点（物种） |
| **优势** | 整合不同精度层级的证据，高层级间接证据也被利用 |
| **vs 简单投票** | RTL 考虑分类树结构，避免忽略属级k-mer的间接支持 |
