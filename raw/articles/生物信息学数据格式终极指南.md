# 生物信息学数据格式终极指南

**作者**：lulu | 宏序生物

---

## 导读

刚入门生物信息学，你是否曾被纷繁复杂的数据格式搞得头晕眼花？SAM、BAM、VCF、GFF……这些字母组合背后究竟藏着什么秘密？今天，我们用一篇文章为你彻底理清思路！

---

## 为什么需要这么多格式？

生物信息学数据具有**多维性、大规模、结构复杂**的特点：

- 原始序列需要保留质量和标识信息
- 比对结果需记录位置、匹配度、链方向
- 基因组注释需结构化存储特征信息
- 变异数据需记录样本基因型、频率等

不同的分析阶段、不同平台产生了专门优化的格式，就像我们根据需求选择 Word、Excel 或 PDF 一样。

---

## 一、测序数据序列格式：FASTA 与 FASTQ

生物信息学中最基础且使用频率最高的两种格式，分别适用于存储简单序列信息和高质量的测序数据。

### FASTA 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | FASTA（生物信息学基础序列格式） |
| **扩展名** | `.fasta` 或 `.fa` |
| **用途说明** | 用于存储 DNA、RNA 或蛋白质序列，是 BLAST 搜索、序列比对和基因组组装的基础格式 |
| **结构说明** | 由两个主要部分组成：以 `>` 开头的描述行和随后的序列行。描述行包含序列标识符和可选信息，序列行包含实际的核苷酸或氨基酸序列 |

### FASTQ 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | FASTQ（高通量测序质量格式） |
| **扩展名** | `.fastq` 或 `.fq` |
| **用途说明** | 专门用于存储测序仪产生的原始读数及其质量分数，是质量控制、序列比对和变异调用的起点 |
| **结构说明** | 每个序列条目由四行组成：① 以 `@` 开头的标识符行 ② 序列行 ③ 以 `+` 开头的分隔行 ④ 质量得分行 |

## 二、序列比对格式：SAM / BAM

当我们将测序数据比对到参考基因组后，就需要一种标准格式来存储比对结果。

### SAM 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | Sequence Alignment/Map（序列比对/映射格式） |
| **扩展名** | `.sam` |
| **用途说明** | 用于存储序列比对结果，包括位置、映射质量和 CIGAR 字符串等信息。是分析基因表达、变异检测和甲基化模式的关键输入 |
| **结构说明** | 纯文本格式，包含：① 头部（以 `@` 开头的元数据行）② 对齐部分（实际的比对数据） |

### BAM 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | Binary Alignment/Map（二进制序列比对格式） |
| **扩展名** | `.bam` |
| **用途说明** | 高效存储和处理大规模比对数据，几乎所有下游分析都使用 BAM 格式。可以看作是 SAM 的"压缩"版本，内容相同但更高效 |
| **结构说明** | 与 SAM 具有相同的结构和信息，但采用二进制压缩，文件更小、处理速度更快 |

## 三、基因组注释格式：GFF / GTF / BED

### GFF 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | General Feature Format（通用特征格式） |
| **扩展名** | `.gff` 或 `.gff3` |
| **用途说明** | 用于描述基因组特征和注释的标准格式，存储基因位置、外显子-内含子结构、非编码 RNA 等。是基因组注释、基因表达量化和功能分析的基础 |
| **结构说明** | 以制表符分隔的字段记录特征信息，结构化存储基因组注释 |

### GTF 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | Gene Transfer Format（基因转移格式） |
| **扩展名** | `.gtf` |
| **用途说明** | 与 GFF 相似，但主要关注基因和转录本结构。用于基因组注释分析 |
| **结构说明** | 与 GFF 相似但有特定区别，主要用于精确描述基因和转录本的结构信息 |

### BED 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | Browser Extensible Data（浏览器可扩展数据格式） |
| **扩展名** | `.bed` |
| **用途说明** | 用于定义基因组区域，如 ChIP-seq 峰、保守区域或自定义间隔。广泛用于基因组浏览器可视化和区间运算 |
| **结构说明** | 简洁的格式，主要用于定义基因组区间，以染色体位置和数值轨道形式存储 |

---

## 四、遗传变异格式：VCF / BCF

### VCF 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | Variant Call Format（变异调用格式） |
| **扩展名** | `.vcf` |
| **用途说明** | 记录基因组中遗传变异（SNP、InDel、SV）的标准格式。是群体遗传学、GWAS 研究和临床遗传学分析的核心格式 |
| **结构说明** | 包含三部分：① 元数据行（以 `##` 开头）② 标题行（以 `#` 开头）③ 数据行（每个数据行对应一个变异位置） |

### BCF 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | Binary Variant Call Format（二进制变异调用格式） |
| **扩展名** | `.bcf` |
| **用途说明** | VCF 的二进制版本，提供更高效的存储和访问，特别适用于多样本分析 |
| **结构说明** | 与 VCF 信息相同，但采用二进制存储，处理更高效 |

## 五、分子结构格式：PDB / mmCIF

### PDB 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | Protein Data Bank（蛋白质数据库格式） |
| **扩展名** | `.pdb` |
| **用途说明** | 蛋白质结构信息的标准表示方法，用于存储和交换大分子三维结构数据。是结构生物学、药物设计和分子对接研究的基础 |
| **结构说明** | 包含：① 原子坐标 ② 实验信息 ③ 二级结构分配 ④ 连接性数据 |

### mmCIF 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | Macromolecular Crystallographic Information File（大分子晶体学信息文件） |
| **扩展名** | `.cif` |
| **用途说明** | PDB 格式的现代化替代与发展，是 PDB 数据库当前官方推荐的全量数据归档与分发格式。用于存储包括超大复合物（核糖体、病毒衣壳）在内的所有大分子结构数据 |
| **结构说明** | 采用基于字典的、机器可读性更强的结构化数据模型，以表格形式的键值对存储信息，克服了传统 PDB 格式在字段长度、数据复杂性和标准化方面的限制 |

## 六、进化分析格式：PHYLIP / Newick

### PHYLIP 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | Phylogeny Inference Package（系统发育推断包格式） |
| **扩展名** | `.phy` 或 `.phylip` |
| **用途说明** | 用于存储系统发育分析所需的多序列比对数据。是进化分析、分子钟研究和分类学研究的基础 |
| **结构说明** | 有两种版本：① 严格版本：所有序列长度相同且连续排列 ② 交错版本：序列分段排列 |

### Newick 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | Newick 树格式 |
| **扩展名** | `.nwk` 或 `.newick` |
| **用途说明** | 以简单的括号表示法描述系统发育树结构，用于表示系统发育树拓扑结构和分支长度。是进化分析结果的可视化和进一步分析的基础 |
| **结构说明** | 如 `(A:0.1,B:0.2,(C:0.3,D:0.4):0.5);` 的括号表示法 |

---

## 七、高效存储格式：HDF5 / mzML

### HDF5 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | Hierarchical Data Format version 5（分层数据格式版本 5） |
| **扩展名** | `.h5` 或 `.hdf5` |
| **用途说明** | 灵活、高效的数据模型，用于存储大型、复杂数据集，如单细胞 RNA-seq 数据、图像数据等需要高效存储和检索的多维数据。可以看作是一种科学数据的"文件系统" |
| **结构说明** | 分层数据结构，支持复杂多维数据的组织和高效访问 |

### mzML 格式

| 项目 | 内容 |
|------|------|
| **格式全称** | Mass Spectrometry XML（质谱数据 XML 格式） |
| **扩展名** | `.mzML` |
| **用途说明** | 质谱数据的开放 XML 格式，是 mzData 和 mzXML 的继承者。用于存储质谱原始数据和元数据，是蛋白质组学、代谢组学和脂质组学分析的标准格式 |
| **结构说明** | 包含质谱仪输出、仪器设置和数据处理步骤的完整描述 |

## 使用建议

根据分析任务选择合适格式：

| 分析任务 | 推荐格式 |
|----------|----------|
| 原始序列处理 | FASTQ |
| 序列存储交换 | FASTA |
| 序列比对分析 | SAM / BAM |
| 基因组注释 | GFF / BED |
| 遗传变异分析 | VCF / BCF |
| 分子结构分析 | PDB / mmCIF |
| 进化分析 | PHYLIP / Newick |
| 大规模数据 | HDF5 / mzML |

---

## 结语

本文聚焦于最通用核心的生物信息学格式。值得注意的是，每个子领域都有其特色格式：

- 进化基因组学使用 **MAF** 分析多物种比对
- 蛋白质组学数据分析流涉及 **mzIdentML / mzQuantML** 等一系列格式
- **BigWig / BigBed** 和 **CRAM** 代表了大规模数据可视化与存储的最新高效格式

掌握核心格式后，可根据研究兴趣深入特定领域。
